便利コマンド

  1. docker-machineのコマンド 参考

    1. ホストマシンの作成（ホスト自体も作成）

      docker-machine create --driver virtualbox [ホスト名]

    2. 既存のSVにdockerを入れて、docker-machine配下に

      docker-machine -D create --driver generic --generic-ip-address [host server ip address] --generic-ssh-port [ssh port] --generic-ssh-user [ssh user] [host name]

    3. ホストの一覧表示

      docker-machine ls

    4. ホスト停止

      docker-machine stop [host name]

    5. ホスト強制停止

      docker-machine kill [host name]

    6. ホスト起動

      docker-machine start [host name]

    7. ホスト再起動

      docker-machine restart [host name]

    8. ホスト切り替え

      docker-machine active [host name]

    9. ホスト削除

      docker-machine rm [host name]

    10. dockerクライアント用の環境変数確認

      docker-machine config [host name]
or
docker-machine $(docker-machine config [host_name]) ps

    11. docker関連の環境変数確認

      env | grep DOCKER

    12. ホストのdockerにマウント

      eval $(docker-machine env [host name])

    13. ホストのdockerマウント解除

      eval $(docker-machine env -u)

    14. ホストマシンのIP表示

      docker-machine ip [host name]

    15. ホストマシンの情報表示

      docker-machine inspect [host name]

    16. ホストへssh接続

      docker-machine ssh [host name]

    17. sshでコマンド実行

      docker-machine ssh [host name] "command"
docker-machine ssh [host name] -- [command] [option]

    18. 既にあるdockerホストをdocker-machineの管理下に

      docker-machine create --url tcp://[ipaddress]:2375 [host name]

    19. アクティブなdockerホストの確認

      echo $DOCKER_HOST

  2. docker-composeコマンド

    1. コンテナ立ち上げ

      docker-compose -f docker-compose.yml up -d
docker-compose -f docker-compose.yml up -d --build
docker-compose -f docker-compose.yml up -d --build --force-recreate

  3. dockerコマンド

    1. コンテナのログ確認

      docker logs [container name]

    2. ネットワーク一覧の表示

      docker network list

    3. ネットワークの詳細

      docker network inspect [network name]

    4. ネットワークの作成

      docker network create --driver bridge [ネットワーク名]

    5. ネットワークの削除 参考

      docker network disconnect -f [network name] [conteniar name]

    6. コンテナの中に入る

      docker exec -i -t [コンテナ名] bash

    7. コンテナ一覧表示

      docker ps -a

    8. コンテナ全削除

      docker rm --force $(docker ps -aq)

  4. alpine linux

    1. apkの更新

      apk update

    2. アプリをまとめてインストール

      apk --no-cache add --virtual build-dependencies curl build-base

    3. 削除

      apk del --purge build-dependencies

    4. インストール済みのものを確認

      apk info

    5. リポジトリ確認

      cat /etc/apk/repositories

    6. 最新のリポジトリを追加

      {
    echo @edge http://nl.alpinelinux.org/alpine/edge/main
    echo @edgecommunity http://nl.alpinelinux.org/alpine/edge/community
    echo @testing http://nl.alpinelinux.org/alpine/edge/testing
} >> /etc/apk/repositories
apk update
apk add --upgrade apk-tools@edge
apk update

    7. edgeのリポジトリからインストール

      apk --no-cache add elasticsearch@edgecommunity=6.1.1-r0

    8. パッケージがインストールされた場所の確認

      apk info -L [package name]

  5. その他

    1. centosのfirewallにdocker接続用のポートを開ける

      sudo firewall-cmd --zone=public --add-port=2376/tcp --permanent
sudo firewall-cmd --reload

    2. テキストファイルの中身をコンソール上に出力

      cat [file name]

    3. コマンドの結果をテキストに上書き出力

      [command] > [file name]
[command] 1> [file name] # 標準出力（デフォルト）
[command] 2> [file name] # エラー出力
[command] &> [file name] # 標準出力とエラー出力を両方

    4. コマンドの結果をテキストに追加出力

      [command] >> [file name]

    5. ホスト上のディレクトリをコピー

      scp -r -P [port] [user]@[host name]:[target path] [outpu path]

    6. nginxの設定を確認

      cat /etc/nginx/conf.d/default.conf
